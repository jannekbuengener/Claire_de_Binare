# WORKFLOW_Supervisor_Playbook (Optimiert)

## Zweck
Standardisierte Führung jeder Aufgabe durch Claude als Supervisor-Orchestrator.

## Rollen
- Hauptmodell (Supervisor & Orchestrator)  
- system-architect  
- devops-engineer  
- test-engineer  
- risk-architect  
- documentation-engineer  
- code-reviewer  

## Phase 0 – Session Start
- Scope klären  
- Risiko-Level bestimmen  
- Analyse-Modus setzen  
- Logging starten  

## Phase 1 – Analyse
- Kontext sammeln  
- Files lesen  
- Risiken identifizieren  
- Fachagenten bei Bedarf aktivieren  
- Analyse-Report erzeugen  

## Phase 2 – Delegation
- minimale Fachagenten aktivieren  
- klar definierte Aufgabenpakete  
- Claude konsolidiert Agenten-Ausgaben  

## Phase 3 – Delivery (User-Go)
- Branch + PR  
- Agenten führen Mutationen aus  
- Claude überwacht Risiko-Gates  

## Phase 4 – Review & Abschluss
- PR-Qualität prüfen  
- Risiken validieren  
- kurzes User-Update  
- Logging + DECISION_LOG
