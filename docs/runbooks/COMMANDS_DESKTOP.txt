# GitHub Desktop – Schrittfolge

1. Application öffnen → „Fetch origin“.
2. „Current Branch“ → „New Branch…“ → Name `feature/cleanroom-dryrun`.
3. Dateien in Explorer auswählen → per Drag & Drop auf GitHub Desktop (Ordner docs/).
4. `.gitignore` in Editor öffnen → Patch aus Runbook einfügen → speichern.
5. Zurück in GitHub Desktop: Review changes → Summary: "chore(cleanroom): add plans + gitignore (dry run)".
6. „Commit to feature/cleanroom-dryrun“ klicken.
7. „Publish branch“ (Push).
8. Button „Create Pull Request“ → öffnet Browser.
9. PR-Formular:
   - Titel: `chore(cleanroom): prepare structure & policy dry run`
   - Body aus `docs/PR_CLEANROOM_DRYRUN.md` kopieren.
   - Labels setzen (`cleanup`, `documentation`, `security-prep`), Milestone wählen.
   - Reviewer hinzufügen.
10. Submit PR → Screenshot für Beweisdokumentation.

## Security-Sanitation via Desktop
1. Externes Terminal/Shell starten (filter-repo/BFG). GitHub Desktop zum Status-Abgleich verwenden.
2. Nach Force-Push: GitHub Desktop → „Fetch origin“ → Bestätigung, dass Branch aktualisiert wurde.

## Dokumentations-PR
1. Nach Moves → „Summary: docs(cleanroom): consolidate documentation tree“ → Commit.
2. Push & PR via Desktop analog.
3. Link-Checker-Lauf in Terminal dokumentieren.
