Du bist Copilot in der CLI im lokalen Repo (Windows PowerShell). Du sollst AUSFÜHREN, nicht diskutieren.

ZIEL:
Erstelle die Quick-Win-PRs (auto-label, emoji-bot checkout-fix, gemini graceful skip, optional labels sync) deterministisch.

HARD RULES (NO EXCUSES):
- Keine Rückfragen. Keine Alternativen. Keine Erklärtexte.
- Wenn ein Schritt nicht möglich ist: gib NUR den nächsten notwendigen Befehl aus und stoppe.
- NIEMALS verwenden: git add -A, git commit -a, git reset --hard, git clean, git stash pop.
- Nur whitelisted Pfade committen (je PR exakt die Dateien, die zu dieser PR gehören).
- Wenn working tree nicht sauber: STOP und gib NUR `git status --porcelain` + den minimalen Fix aus.

VORGEHEN:
1) Preflight ausführen:
   - git rev-parse --show-toplevel
   - git branch --show-current
   - git fetch origin
   - git checkout main
   - git pull --ff-only origin main
   - git status --porcelain
   - gh auth status || gh auth login --web

2) Erstelle im Repo-Root eine Datei `create_prs_safe.ps1` (PowerShell), die ALLES automatisiert:
   - Erstellt pro PR einen Branch
   - Schreibt/ändert nur die dafür nötigen Dateien
   - `git add` NUR diese Dateien (kein add -A)
   - commit, push, `gh pr create`
   - Am Ende: `gh pr list --state open --limit 20 --json number,url,title` ausgeben

3) Das Script muss diese PRs erstellen (wenn Branch/PR schon existiert: skip und weiter):
   A) chore/infra/auto-label-from-branch
      Dateien: .github/workflows/auto-label-from-branch.yml, .github/workflows/labels.json, .github/BRANCH_LABELING.md, PR_BODY_auto_label.md
   B) chore/infra/emoji-bot-checkout-fix
      Dateien: .github/workflows/emoji-bot.yml, PR_BODY_emoji.md
      Wichtig: KEIN git add -A in workflow steps (wenn vorhanden: entfernen).
   C) chore/infra/gemini-graceful-skip
      Dateien: .github/workflows/gemini-triage.yml, .github/workflows/gemini-review.yml, PR_BODY_gemini.md
      Guard: graceful skip wenn required secrets/vars fehlen (exit 78, klare Logs).
   D) chore/infra/sync-labels (nur wenn labels.json NICHT bereits in A enthalten ist oder sich unterscheidet)
      Dateien: .github/workflows/labels.json, PR_BODY_labelsync.md

4) Nachdem `create_prs_safe.ps1` geschrieben wurde: führe es aus:
   - powershell -ExecutionPolicy Bypass -File .\create_prs_safe.ps1

OUTPUT (am Ende, sonst nichts):
- Liste der erstellten PR-URLs (eine pro Zeile)
- Danach: gh pr list Ausgabe

JETZT LOS: führe Schritt 1 aus, dann schreibe Schritt 2-4 und führe es aus.
